InfoCaption Dashboard

En modulär verktygsplattform för InfoCaptions supportorganisation med verktyg för kundanalys, e-postutskick, serverövervakning, certifikathantering, SQL-frågor, kodning och mycket mer. Enkel att använda och utbyggbar med egna moduler.

Tips! Använd sidomenyn till vänster för att navigera mellan modulerna eller klicka på modulkorten på startsidan. Alla hanteringssidor (Grupper, Användare, Admin) laddas direkt i dashboarden.

Kundstatistik

Snabbstart

Lär dig grunderna i InfoCaption Dashboard på några minuter.

Navigera i dashboarden

Dashboarden består av fyra huvuddelar:

  1. Sidopanelen (vänster) — Navigation till alla moduler och hanteringssidor
  2. Widget-baren (överst) — Kompakta data-widgets (klocka, serverstatus, certifikat, etc.)
  3. Topbaren — Breadcrumb-navigation och modulens namn
  4. Innehållsområdet — Där den aktiva modulen visas i en iframe

Använda moduler

För att öppna en modul:

  1. Klicka på modulens namn i sidopanelen, eller
  2. Klicka på modulkortet på startsidan

Hanteringssidor

Under rubriken "Hantera" i sidomenyn finns:

  • Skapa modul — Ladda upp nya HTML-moduler
  • Mina moduler — Redigera, ta bort och ändra gruppsynlighet
  • Grupper — Gå med/lämna grupper, hantera medlemmar
  • Användare — Se alla användare, starta Teams-chatt
  • Admin — Adminpanel (kräver admin-rättigheter)

Alla hanteringssidor laddas direkt i dashboardens iframe — du lämnar aldrig huvudsidan.

Bra att veta! Du kan dölja sidopanelen genom att klicka på menyknappen (☰) i topbaren för att få mer utrymme.

URL-parametrar

Du kan länka direkt till en specifik modul eller hanteringssida:

dashboard.jsp?module=customer-stats
dashboard.jsp?module=utskick
dashboard.jsp?manage=manage-groups
dashboard.jsp?manage=admin

Modulens -knapp

Varje modul har en flytande -knapp (nere till höger) som öppnar en dokumentationspanel med modulens beskrivning och AI-specifikation.

Kundstatistik

Analysera och visualisera kundstatistik med automatisk flaggning av risk- och succékunder.

Funktioner

  • Succékunder-analys med anpassningsbara kriterier
  • Riskkunder-flaggning baserat på trendanalys
  • Historikdiagram per kund (dag/månad, datumväljare)
  • Tabell- och kortvy med växlingsknapp
  • Excel-export med SheetJS
  • Anpassningsbara risk-/succétröskelvärden

Datakälla

Statistik samlas in per server (URL) via PowerShell-skriptet allstats.ps1 som körs dagligen. API:et /api/customer-stats aggregerar data i 30-dagarsperioder. Diagram hämtar historik via /api/customer-stats/history.

Riskkriterier

En kund flaggas som risk om något av följande stämmer:

PrioKriteriumBeskrivning
1Trend: Visningar ↓Minskning ≥30% (30d vs föregående 30d)
2Ingen inloggning<1 inloggning senaste månaden
3InloggningskollapsInloggningar <20% av totala
4Trend: Guider ↓Minskning ≥30%
5Låg publicering<10 publiseringar senaste månaden
6Trend: Processer ↓Minskning ≥30%

Succékriterier

En kund räknas som succé om alla villkor uppfylls:

PrioKriteriumBeskrivning
1Trend: Visningar ↑Ökning ≥30%
2Hög inloggning>100 inloggningar senaste månaden
3Trend: Guider ↑Ökning ≥30%
4Trend: Unika producenter ↑Ökning ≥20%
5Hög publicering>100 publiseringar senaste månaden
6Trend: Processer ↑Ökning ≥30%

Utskick

Skicka e-post till mottagarlistor med mallar, variabelstöd och kontaktlistor från extern databas.

Flikar

  • Nytt utskick — Välj mall, redigera HTML, ange mottagare, skicka
  • Listor — Bygg mottagarlistor från kontaktdatabasen med filter (server, domän, företag, språk)
  • Mallar — Skapa, redigera och ta bort e-postmallar
  • Historik — Se tidigare utskick med mottagarstatus per e-post

Mallvariabler

Följande variabler ersätts per mottagare vid sändning:

VariabelBeskrivning
{{serverUrl}}Mottagarens server-URL (från Listor)
{{email}}Mottagarens e-postadress
{{companyName}}Företagsnamn (från Listor)
{{updateDate}}Globalt datumfält i sändformuläret

Listor — Filter

Bygg mottagarlistor genom att filtrera den externa kontaktdatabasen:

  • ServerUrl, MachineName, Domain (fritext)
  • CompanyName, CompanyCategory, PersonCategory
  • Language — resultaten visar språkfördelning med per-språk "Utskick"-knapp
Tips! Klicka "Använd i utskick" på en lista för att föra över alla e-postadresser till sändfliken. Du kan filtrera per språk för att skicka språkanpassade utskick.

SQL Builder

Bygg SQL-frågor enkelt med variabler. Stöd för MySQL och SQL Server.

Funktioner

Använda variabler

Variabler definieras med syntaxen ${variabelnamn} i SQL-frågan:

SELECT * FROM users
WHERE company_id = ${companyId}
AND created_date >= '${startDate}'

Grupper

Gruppbaserad åtkomstkontroll styr vilka moduler och widgets som visas för varje användare.

Grundkoncept

  • "Alla"-gruppen — Alla användare är implicita medlemmar. Moduler tilldelade "Alla" syns för alla.
  • Synliga grupper — Användare kan själva gå med/lämna via Grupper-sidan.
  • Dolda grupper — Syns inte för icke-medlemmar, kan inte självjoinas. Medlemmar hanteras av admin.
  • Modulfiltrering — En modul syns om du tillhör någon av dess grupper. Moduler utan grupp syns för alla.

SSO-avdelningsgrupper

Grupper kan kopplas till SSO-avdelningar. Vid SSO-inloggning tilldelas användare automatiskt till grupper som matchar deras avdelning. Stöd för pipe-separerade multi-avdelningar (t.ex. Utveckling|Kundvård). Se SSO & Inloggning-sektionen.

Hantera grupper

Under Hantera → Grupper kan du:

  • Se "Mina grupper" och "Tillgängliga grupper"
  • Gå med/lämna synliga grupper
  • Se och hantera medlemmar

Admins kan hantera alla grupper i Admin → Grupper.

Administration

Adminpanelen ger åtkomst till systemkonfiguration och användarhantering.

Admin-rättigheter krävs. Kontakta en befintlig admin om du behöver åtkomst.

Flikar

1. Användare

Hantera användarkonton:

  • Se alla användare med namn, e-post, senaste inloggning
  • Toggla admin-status per användare
  • Ta bort användare
  • Sök och filtrera

2. Grupper

Hantera grupper och SSO-koppling:

  • Skapa/redigera/ta bort grupper
  • Koppla grupp till SSO-avdelning
  • Hantera medlemmar (lägg till/ta bort)
  • Toggla dold/synlig

3. Datasynk

Konfigurera automatisk dataimport från externa API:er:

  • Steg-för-steg wizard: URL → Test → Fältmappning → Schema
  • Stöd för API-nyckel, Bearer, Basic-autentisering
  • FK-lookup (t.ex. company_name → customer_id)
  • Manuell eller schemalagd körning (5/10/30/60 min)
  • Körhistorik med felmeddelanden

4. Widgets

Hantera dashboard-widgets:

  • Se alla widgets (inbyggda + anpassade)
  • Skapa anpassade widgets med HTML/JS
  • Grupptilldelning

5. Inställningar

Centraliserad konfiguration för hela systemet:

  • Alla nycklar grupperade: Databas, E-post, Säkerhet, Allmänt
  • Hemliga värden maskerade (lösenord, API-nycklar)
  • Sök och filtrera
  • Ändringar gäller direkt (5 min cache)

SSO & Inloggning

SAML2 Single Sign-On med Microsoft Entra ID och konfigurerbar inloggning.

Inloggningsmetoder

Konfigureras via auth.loginMethods i Admin → Inställningar:

VärdeBeteende
bothLösenord + SSO-knapp (standard)
passwordBara lösenord
ssoBara SSO — auto-redirect till Entra ID

SSO-flöde

  1. Användaren klickar "Logga in med Microsoft"
  2. Redirect till Microsoft Entra ID
  3. Microsoft autentiserar och returnerar SAML-assertion
  4. Dashboarden skapar/uppdaterar användarkonto automatiskt
  5. Om sso.autoAssignGroups=true: synkar gruppmedlemskap baserat på avdelning

Automatisk grupptilldelning

SSO-inloggning kan automatiskt tilldela användare till grupper baserat på deras avdelning:

  • Läser department-claim från SAML-assertion
  • Stöd för pipe-separerade avdelningar: Utveckling|Kundvård
  • 3-stegs matchning: sso_department → gruppnamn → auto-skapa
  • Gamla SSO-grupper rensas — synkas till aktuella avdelningar

Konfigurerbara claim-URI:er

Alla SAML claim-URI:er kan ändras i Admin → Inställningar utan kodändring:

  • sso.emailClaimUri — E-postadress
  • sso.givenNameClaimUri — Förnamn
  • sso.surnameClaimUri — Efternamn
  • sso.departmentClaimUri — Avdelning

Skapa nya moduler

Skapa och registrera nya moduler i InfoCaption Dashboard.

Fullständig dokumentation: Se docs/MODULE-GUIDE.md för detaljerad guide och docs/MODULE-SPEC.md för teknisk specifikation (AI-vänlig).

Via webben (enklaste sättet)

  1. Klicka Skapa modul i sidomenyn
  2. Fyll i namn, ikon (emoji), beskrivning, kategori
  3. Välj synlighet: Privat eller Delad (med gruppval)
  4. Ladda upp HTML-fil eller ZIP-arkiv (max 50MB)
  5. Skriv valfri AI-specifikation och generera AI-prompt
  6. Klicka Skapa

Manuellt (för utvecklare)

  1. Skapa mapp: webapp/modules/min-modul/
  2. Lägg HTML-fil i mappen
  3. Registrera i modules-tabellen i MySQL
INSERT INTO modules (module_type, name, icon, description, entry_file, directory_name)
VALUES ('system', 'Min Modul', '🚀', 'Beskrivning', 'index.html', 'min-modul');

Vanliga frågor

Svar på vanliga frågor om InfoCaption Dashboard.

Allmänt

Vilka webbläsare stöds?

Alla moderna webbläsare: Chrome, Firefox, Edge och Safari. Vi rekommenderar senaste versionen.

Hur loggar jag in?

Antingen med lösenord eller via "Logga in med Microsoft" (SAML2 SSO). Vilka metoder som visas beror på konfigurationen (auth.loginMethods).

Varför ser jag inte alla moduler?

Moduler kan vara begränsade till specifika grupper. Gå till Hantera → Grupper för att se tillgängliga grupper och gå med.

Kundstatistik

Var kommer kunddata ifrån?

Statistik samlas in dagligen per server via PowerShell-skriptet allstats.ps1 och lagras i databasen. API:et aggregerar data i 30-dagarsperioder.

Varför ser jag inte alla kunder?

Kunder med färre än 5 producenter exkluderas automatiskt. Tröskelvärdet kan justeras i kriterier-inställningarna ().

Utskick

Varifrån skickas e-posten?

Via Azure Communication Services. Avsändaradress konfigureras i Admin → Inställningar (acs.senderAddress).

Hur bygger jag en mottagarlista?

Använd Listor-fliken i Utskick-modulen. Filtrera på server, domän, företag, språk och klicka "Använd i utskick".

Administration

Hur blir jag admin?

En befintlig admin kan ge dig admin-rättigheter via Admin → Användare. Första admin sätts via SQL: UPDATE users SET is_admin = 1 WHERE id = 1;

Hur kopplar jag en grupp till en SSO-avdelning?

I Admin → Grupper, redigera gruppen och fyll i fältet "SSO-avdelning (department)" med exakt det värde som Azure skickar.

Var ändrar jag inställningar?

Admin → Inställningar. Alla konfigurationsnycklar (databas, e-post, SSO, timeouts) kan ändras där. Ändringar gäller inom 5 minuter (cache).
